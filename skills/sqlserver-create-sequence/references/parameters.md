# SQL Server CREATE SEQUENCE — Parameter Reference

## Full Syntax

```sql
CREATE SEQUENCE [ schema_name . ] sequence_name
    [ AS [ built_in_integer_type | user-defined_integer_type ] ]
    [ START WITH <constant> ]
    [ INCREMENT BY <constant> ]
    [ { MINVALUE [ <constant> ] } | { NO MINVALUE } ]
    [ { MAXVALUE [ <constant> ] } | { NO MAXVALUE } ]
    [ CYCLE | { NO CYCLE } ]
    [ { CACHE [ <constant> ] } | { NO CACHE } ]
[ ; ]
```

## Parameters

### AS data_type
- The integer data type for the sequence values.
- Supported types: `TINYINT`, `SMALLINT`, `INT`, `BIGINT`, `DECIMAL` (scale 0), `NUMERIC` (scale 0).
- Default: `BIGINT`.

```sql
AS INT          -- 32-bit integer range
AS BIGINT       -- 64-bit integer range (default)
AS SMALLINT     -- 16-bit integer range
AS DECIMAL(10)  -- Up to 10 digits, no decimal places
```

### START WITH
- The first value generated by the sequence.
- Default: `MINVALUE` for ascending sequences; `MAXVALUE` for descending sequences.

```sql
START WITH 1        -- Start at 1
START WITH 1000     -- Start at 1000
START WITH -100     -- Start at negative value
```

### INCREMENT BY
- The value added to the previous sequence value to generate the next.
- Positive value: ascending sequence.
- Negative value: descending sequence.
- Cannot be 0.
- Default: 1.

```sql
INCREMENT BY 1      -- Standard incrementing sequence
INCREMENT BY 5      -- Increment by 5 (e.g., for batch sizing)
INCREMENT BY -1     -- Descending sequence
INCREMENT BY 10     -- Skip values
```

### MINVALUE / NO MINVALUE
- The minimum value for a cyclic sequence or starting point for descending sequences.
- `NO MINVALUE` (default): uses the minimum value of the data type.

```sql
MINVALUE 1           -- Minimum is 1 (common for ascending sequences)
MINVALUE 100         -- Skip low values
NO MINVALUE          -- Default: minimum of data type (e.g., -2^63 for BIGINT)
```

### MAXVALUE / NO MAXVALUE
- The maximum value for the sequence.
- `NO MAXVALUE` (default): uses the maximum value of the data type.

```sql
MAXVALUE 999999      -- Wrap or stop at 999999
MAXVALUE 2147483647  -- INT max value
NO MAXVALUE          -- Default: maximum of data type
```

### CYCLE / NO CYCLE
- `CYCLE`: After reaching MAXVALUE (ascending) or MINVALUE (descending), the sequence restarts from MINVALUE (ascending) or MAXVALUE (descending).
- `NO CYCLE` (default): After reaching the limit, further requests raise an error.

```sql
CYCLE       -- Wrap around after reaching the boundary
NO CYCLE    -- Error if boundary exceeded (default, safer)
```

### CACHE / NO CACHE
- `CACHE n`: Pre-allocates `n` sequence values in memory for performance.
  - Surviving values are lost on SQL Server restart (gaps will appear after restart).
  - Recommended for high-frequency sequence generation.
- `NO CACHE`: Each NEXT VALUE FOR call writes to disk (slower but no gaps on restart).
- Default cache size: implementation-specific (typically 50).

```sql
CACHE 100    -- Cache 100 values in memory
CACHE 1000   -- Larger cache for very high frequency use
NO CACHE     -- No caching, slowest but no gaps on restart
```

## Usage: NEXT VALUE FOR

```sql
-- Get the next value
SELECT NEXT VALUE FOR dbo.seq_invoice_number;

-- Use in an INSERT
INSERT INTO finance.Invoice (InvoiceNumber, CustomerId, InvoiceDate)
VALUES (NEXT VALUE FOR dbo.seq_invoice_number, 42, GETDATE());

-- Generate multiple sequence values in one batch
SELECT TOP 10 NEXT VALUE FOR dbo.seq_invoice_number OVER (ORDER BY (SELECT NULL));
```

## Using Sequence as Column Default

```sql
-- Create the sequence
CREATE SEQUENCE dbo.seq_order_id
AS INT
START WITH 1
INCREMENT BY 1
MINVALUE 1
NO MAXVALUE
NO CYCLE
CACHE 50;
GO

-- Reference sequence in DEFAULT constraint
CREATE TABLE sales.[Order]
(
    OrderId     INT           NOT NULL
                              CONSTRAINT DF_Order_OrderId
                              DEFAULT (NEXT VALUE FOR dbo.seq_order_id),
    CustomerId  INT           NOT NULL,
    OrderDate   DATETIME2     NOT NULL CONSTRAINT DF_Order_OrderDate DEFAULT GETDATE(),
    CONSTRAINT PK_Order PRIMARY KEY CLUSTERED (OrderId)
);
GO
```

## Difference from IDENTITY

| Feature | IDENTITY | SEQUENCE |
|---------|----------|----------|
| Scope | Column-specific | Database object, shareable |
| Get next value | INSERT only | NEXT VALUE FOR anywhere |
| Cross-table use | No | Yes |
| Restart | DBCC CHECKIDENT | ALTER SEQUENCE RESTART |
| Use as default | Implicit | DEFAULT (NEXT VALUE FOR ...) |
| Preview next value | `IDENT_CURRENT` | `sys.sequences.current_value` |
| Gaps after restart | Possible | Possible (with CACHE) |

## Alter and Restart

```sql
-- Change cache size
ALTER SEQUENCE dbo.seq_invoice_number
    CACHE 200;

-- Restart sequence at a new value
ALTER SEQUENCE dbo.seq_invoice_number
    RESTART WITH 10000;

-- Change max value
ALTER SEQUENCE dbo.seq_invoice_number
    MAXVALUE 9999999;
```

## Complete Example

```sql
-- Invoice number sequence: 4-digit zero-padded (formatted in app or function)
CREATE SEQUENCE finance.seq_invoice_number
AS INT
START WITH 1
INCREMENT BY 1
MINVALUE 1
MAXVALUE 2147483647
NO CYCLE
CACHE 100;
GO

-- Use in table
CREATE TABLE finance.Invoice
(
    InvoiceId       INT            NOT NULL IDENTITY(1,1),
    InvoiceNumber   INT            NOT NULL
                                   CONSTRAINT DF_Invoice_InvoiceNumber
                                   DEFAULT (NEXT VALUE FOR finance.seq_invoice_number),
    CustomerId      INT            NOT NULL,
    InvoiceDate     DATE           NOT NULL,
    DueDate         DATE           NOT NULL,
    TotalAmount     DECIMAL(18,2)  NOT NULL,
    CONSTRAINT PK_Invoice PRIMARY KEY CLUSTERED (InvoiceId),
    CONSTRAINT UQ_Invoice_InvoiceNumber UNIQUE (InvoiceNumber)
);
GO
```

## System Views for Inspection

```sql
-- List all sequences
SELECT s.name AS SchemaName, seq.name AS SequenceName,
       TYPE_NAME(seq.system_type_id) AS DataType,
       seq.start_value, seq.increment, seq.minimum_value,
       seq.maximum_value, seq.is_cycling, seq.is_cached,
       seq.cache_size, seq.current_value
FROM sys.sequences seq
JOIN sys.schemas s ON seq.schema_id = s.schema_id
ORDER BY s.name, seq.name;
```
